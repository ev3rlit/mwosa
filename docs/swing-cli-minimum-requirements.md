# Swing CLI Minimum Requirements

## 목적

이 문서는 스윙 매매 보조를 위한 CLI의 최소 기능요건을 정리한다.

현재 저장소는 구현보다 문서 기준을 먼저 고정하는 단계이므로, 이 문서는 향후 Go 기반 CLI가 어떤 사용자 흐름을 반드시 제공해야 하는지 정의하는 기준으로 사용한다.

## 범위

이 문서의 범위는 다음 5개 흐름이다.

- 스크리너
- 진입 체크리스트
- 포지션 트래커
- 매매일지
- 주간 통계

핵심 원칙은 간단하다.

- 일별 시세와 현재가 스냅샷 같은 canonical data 위에서 동작해야 한다.
- 사람이 바로 읽을 수 있는 출력과 기계가 다시 처리할 수 있는 구조화 출력 둘 다 가능해야 한다.
- 자동매매나 주문 실행보다, 규칙 기반 의사결정과 기록 보존을 우선한다.

## 최소 기능요건

### 1. 스크리너

CLI는 스윙 후보를 좁히기 위한 기본 스크리닝 명령을 제공해야 한다.

필수 필터:

- `RV`(상대거래량)
- `RS`(상대강도)
- 돌파 패턴 필터

최소 동작:

- 지정한 유니버스 또는 심볼 목록을 대상으로 계산할 수 있어야 한다.
- 각 종목에 대해 필터 통과 여부와 핵심 계산값을 함께 반환해야 한다.
- 결과는 정렬 가능해야 하며, 최소한 `RV`, `RS`, 최근 가격, 돌파 여부를 포함해야 한다.
- 후속 흐름에서 재사용할 수 있도록 JSON 출력이 가능해야 한다.

최소 출력 예시 필드:

- `symbol`
- `name`
- `rv`
- `rs`
- `breakout`
- `last_price`
- `as_of`

### 2. 진입 체크리스트

CLI는 진입 전 리스크 검토를 빠르게 끝낼 수 있는 계산 명령을 제공해야 한다.

필수 기능:

- `R/R` 자동 계산
- 포지션 사이즈 역산

최소 동작:

- 사용자가 진입가, 손절가, 목표가, 허용 손실 금액 또는 계좌 리스크 비율을 입력할 수 있어야 한다.
- 시스템은 1주 또는 1계약 기준 손실폭을 계산하고, 허용 손실 한도에 맞는 수량을 역산해야 한다.
- 계산 결과에 총 진입 금액, 예상 손실 금액, 예상 이익 금액, `R/R` 값을 포함해야 한다.
- 계산 근거가 명확하도록 입력값과 계산식을 함께 보여줄 수 있어야 한다.

최소 출력 예시 필드:

- `entry_price`
- `stop_price`
- `target_price`
- `risk_per_unit`
- `reward_per_unit`
- `rr_ratio`
- `max_loss_budget`
- `position_size`
- `notional_amount`

### 3. 포지션 트래커

CLI는 열린 포지션과 종료된 포지션을 모두 기록하고 조회할 수 있어야 한다.

필수 필드:

- 진입일
- 진입시간
- 청산일
- 청산시간
- 손절가
- 1차 익절가

최소 동작:

- 포지션 생성 시 진입 기준 정보와 리스크 기준 정보를 함께 저장해야 한다.
- 열린 포지션은 손절가와 1차 익절가를 기준으로 추적 가능해야 한다.
- 청산 시점에는 청산 일시, 청산가, 청산 수량, 최종 상태를 기록해야 한다.
- 부분 청산이 있더라도 최소한 1차 익절 여부는 추적 가능해야 한다.

최소 출력 예시 필드:

- `position_id`
- `symbol`
- `entry_at`
- `exit_at`
- `entry_price`
- `exit_price`
- `stop_price`
- `take_profit_1_price`
- `quantity`
- `status`

### 4. 매매일지

CLI는 각 포지션에 대해 의사결정 근거와 사후 평가를 남길 수 있어야 한다.

필수 기록 항목:

- 진입 근거
- 청산 사유
- 룰 준수 여부

최소 동작:

- 포지션 단위로 진입 당시 메모를 남길 수 있어야 한다.
- 청산 시점에 청산 사유를 별도로 기록할 수 있어야 한다.
- 사용자가 자신의 룰을 지켰는지 `yes/no` 또는 동등한 명시적 값으로 남길 수 있어야 한다.
- 일지 조회 시 가격 정보와 메모 정보가 함께 연결되어 보여야 한다.

최소 출력 예시 필드:

- `position_id`
- `entry_reason`
- `exit_reason`
- `rule_followed`
- `rule_violation_note`
- `created_at`
- `updated_at`

### 5. 주간 통계

CLI는 주간 단위로 매매 성과와 규칙 준수 수준을 요약할 수 있어야 한다.

필수 지표:

- 승률
- 평균 `R/R`
- 기댓값
- 룰 준수율

최소 동작:

- 지정한 주간 범위 안에서 종료된 거래를 집계할 수 있어야 한다.
- 각 거래의 손익을 `R` 기준과 금액 기준으로 모두 계산할 수 있어야 한다.
- 손절, 익절, 임의 청산 등 종료 사유별 분포를 함께 볼 수 있으면 좋지만, 최소 요건은 아니다.
- 주간 요약은 사람이 읽는 표 형태와 JSON 형태 둘 다 제공할 수 있어야 한다.

최소 출력 예시 필드:

- `week_start`
- `week_end`
- `trade_count`
- `win_rate`
- `average_rr`
- `expectancy_r`
- `rule_follow_rate`

## 데이터 연결 관점의 최소 요구

위 5개 흐름은 서로 분리된 기능이 아니라 하나의 거래 기록 흐름으로 이어져야 한다.

- 스크리너 결과에서 후보 종목을 선택할 수 있어야 한다.
- 진입 체크리스트 계산 결과를 포지션 생성 시 재사용할 수 있어야 한다.
- 포지션 종료 정보는 자동으로 매매일지와 주간 통계에 반영될 수 있어야 한다.
- 동일 거래는 `position_id` 같은 공통 키로 연결되어야 한다.

## 비목표

이 문서에서 아직 최소 요건으로 포함하지 않는 항목은 다음과 같다.

- 자동 주문 실행
- 실시간 알림
- 브로커 계좌 연동
- 고급 백테스트
- 멀티 계좌/멀티 전략 권한 모델

## 구현 우선순위

최소 기능은 다음 순서로 구현하는 것이 자연스럽다.

1. 스크리너
2. 진입 체크리스트
3. 포지션 트래커
4. 매매일지
5. 주간 통계

이 순서는 실제 사용 흐름과 같다. 먼저 후보를 찾고, 진입을 검토하고, 포지션을 추적하고, 거래를 기록한 뒤, 주간 단위로 복기한다.
